# Команды аудита и проверки безопасности (Rust + WASM)

Этот файл содержит готовые терминальные команды, которые ИИ-агент должен использовать для реальной, а не галлюцинированной проверки.

## 1. Проверка крейта перед добавлением

```bash
# Получить информацию о крейте
cargo search <имя_крейта>

# Посмотреть дерево зависимостей (очень важно!)
cargo tree -i <имя_крейта> --duplicates

# Проверить, нет ли у крейта известных уязвимостей (RustSec)
cargo audit
```

## 2. Глубокий аудит после измененений

```bash
# Проверка уязвимостей в Cargo.lock
cargo audit

# Поиск устаревших зависимостей
cargo outdated

# Мощный линтер (лицензии + бан крейтов + дубликаты)
# Установка: cargo install cargo-deny
cargo deny check

# Проверка лицензий в зависимостях
cargo deny check licenses
```

## 3. WASM и финальные артефакты

```bash
# Оптимизация WASM (рекомендуется перед деплоем)
wasm-opt -O3 your_project_bg.wasm -o your_project_bg_optimized.wasm

# Просмотр содержимого WASM бинарника (для проверки на hidden eval и подозрительные вызовы)
wasm-objdump -d your_project_bg.wasm | head -100

# Проверка размера и структуры WASM
wasm-objdump -x your_project_bg.wasm
```

## 4. Дополнительные полезные команды

```bash
# Полное дерево зависимостей проекта
cargo tree --duplicates

# Проверка лицензий всех зависимостей
cargo tree --prefix none | sort | uniq

# Проверка версий toolchain
rustup show
```

## Примечания для закрытого контура

- `cargo audit` работает с локальным зеркалом RustSec при наличии `~/.cargo/advisory-db`.
- Для полностью закрытого контура рекомендуется периодически обновлять базу адвайзори в оффлайн-режиме.
- `cargo deny` можно настраивать под конкретные политики безопасности организации.

---

**Используй эти команды вместо с галлюцинаций.** Агент должен выполнять их реально, а не придумывать результаты.
